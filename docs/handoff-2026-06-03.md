## 5. 운영/스테이징 주의점

현재 `docker-compose.yml`은 로컬 MVP 검증용이다. 외부 체험이나 운영 후보 환경에 그대로 공개하지 않는다.

주의할 값:

- `SPRING_PROFILES_ACTIVE`
- `JWT_SECRET`
- DB 계정과 비밀번호
- `FILE_STORAGE_ROOT`
- seed/test 데이터 적용 여부
- PostgreSQL 외부 포트 공개 여부
- backend 직접 외부 공개 여부
- HTTPS 적용 여부

외부 체험 기본 방향은 작은 스테이징 서버에 단독 staging compose를 올리는 것이다. Cloudflare Tunnel 또는 ngrok은 아주 빠른 임시 공유용 보조안으로만 사용한다.

## 6. 계정과 권한 메모

seed/local profile 기준 기본 비밀번호는 `password`다.

주요 seed 계정:

- `admin/password`: 관리자
- `employee01/password`: 기안자
- `lead-dev/password`: 개발팀 결재자
- `approver01/password`: 기본 결재선 검증용 결재자
- `ceo/password`: `APPROVER,MANAGER` 역할의 매니저 계정

권한 정책:

- 일반 사용자는 관리 메뉴를 볼 수 없고 `/admin/...` 직접 접근 시 `/dashboard`로 이동한다.
- `ADMIN`과 `MANAGER`는 관리 메뉴에 접근할 수 있다.
- 사용자 관리의 민감 CRUD와 타 사용자 비밀번호 변경은 `ADMIN` 중심으로 유지한다.
- 대리결재는 제외하고, 관리자/매니저 예외 결재를 사용한다.
- 관리자 예외 결재 사유는 필수 입력 경로다.
